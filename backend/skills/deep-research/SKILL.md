# Deep Research 深度研究

通过 `web_search` + `web_fetch` 进行多源深度研究，综合分析后输出带引用的结构化报告。使用 `write_file` 保存完整报告到 `output/` 目录。

## 调用方式

使用内置工具组合：
- `web_search(query)` — 搜索互联网，获取结构化结果
- `web_fetch(url)` — 抓取关键网页的完整内容
- `python_executor(code)` — 数据处理与分析（可选）
- `write_file(path, content)` — 保存研究报告

## 研究工作流

### 第 1 步：理解研究目标

快速确认 1-2 个关键点：
- 用户的目标是什么？（学习、决策、写作、汇报）
- 有无特定角度或深度要求？

如果用户说"直接研究"，按合理默认值执行。

### 第 2 步：规划研究框架

将主题拆分为 3-5 个研究子问题。例如：

**主题**：AI 对医疗的影响
- 当前医疗 AI 的主要应用领域有哪些？
- 已有哪些临床效果数据？
- 监管和合规方面的挑战？
- 哪些公司/机构在引领？
- 市场规模和增长趋势？

### 第 3 步：多源搜索

对每个子问题执行搜索：

```
web_search(query="子问题关键词", max_results=8)
```

**搜索策略：**
- 每个子问题使用 2-3 种不同的关键词组合
- 混合使用中文和英文关键词以获取更广泛的视角
- 目标：收集 15-30 个不同来源
- 来源优先级：学术论文、官方报告、权威媒体 > 行业博客 > 论坛

### 第 4 步：深度阅读关键来源

对最有价值的 URL 获取完整内容：

```
web_fetch(url="https://example.com/article")
```

- 精读 3-5 篇关键来源，不仅依赖搜索摘要
- 提取关键数据、引用和论据
- 记录每条重要信息的来源 URL

### 第 5 步：交叉验证

- 关键数据至少需要 2 个独立来源确认
- 仅有单一来源的声明标记为"待验证"
- 优先使用近 12 个月的来源
- 对无法找到充分信息的子问题，明确标注信息不足

### 第 6 步：综合撰写报告

使用以下模板结构：

```markdown
# [主题]：深度研究报告
*生成日期：[date] | 来源数量：[N] | 置信度：[高/中/低]*

## 摘要
[3-5 句话概括核心发现]

## 1. [第一个主要主题]
[分析发现，内联引用]
- 关键发现 ([来源名称](url))
- 支撑数据 ([来源名称](url))

## 2. [第二个主要主题]
...

## 3. [第三个主要主题]
...

## 关键结论
- [可操作的洞察 1]
- [可操作的洞察 2]
- [可操作的洞察 3]

## 参考来源
1. [标题](url) — [一句话摘要]
2. ...

## 研究方法
搜索了 [N] 个查询，分析了 [M] 个来源。
研究子问题：[列表]
```

### 第 7 步：保存与交付

保存完整报告：

```
write_file(path="output/research_[slug].md", content="完整报告内容")
```

交付策略：
- **短报告**：直接在对话中输出完整内容
- **长报告**（>2000 字）：在对话中输出摘要 + 关键结论，提示用户查看完整报告文件

## 质量规则

1. **每条声明都需要来源**。不做无来源的断言。
2. **交叉验证**。仅有一个来源的信息，标记为"待验证"。
3. **时效性**。优先使用近 12 个月内的来源。
4. **承认不足**。对信息不足的子问题，明确说"当前信息不足以得出结论"。
5. **不臆造**。无法确认的信息不编造，标注"未找到充分数据"。

## 研究场景示例

```
"研究 2026 年核聚变能源的最新进展"
"深度对比 Rust vs Go 在后端服务的适用性"
"研究 SaaS 创业的最佳 bootstrapping 策略"
"调研中国新能源汽车出海东南亚的市场机会"
"竞品分析：Notion vs Obsidian vs Logseq"
```

## 结合数据分析

对于需要数据支撑的研究，可组合使用其他 Skills：
- 用 `duckdb-analysis` 进行 SQL 数据查询
- 用 `pandas-analysis` 进行数据可视化和统计分析
- 将图表和数据表格嵌入研究报告

## 环境说明

- 报告保存到 `output/` 目录（相对于 backend 工作目录）
- `web_search` 和 `web_fetch` 为内置工具，无需额外 API Key 配置
